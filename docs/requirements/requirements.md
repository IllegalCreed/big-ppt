# Lumideck 功能需求

> 本文档定位：**功能/需求点头脑风暴记录**。颗粒度 = "用户能做什么 / 系统应支持什么"，不写实现方式。
>
> 工程信息见 [`CLAUDE.md`](../../CLAUDE.md)；产品形态与商业畅想见 [`vision.md`](./vision.md)；分阶段排期见 [`roadmap.md`](./roadmap.md)。
>
> **不在本文写 Phase 编号** — Phase 编号在 roadmap 唯一管理，避免编号 drift 时同步。

## 功能需求

### FR-01 模板系统

- 支持多套模板，每套包含封面、封底、目录页、章节标题页、内容页等多种页面类型
- 模板与代码分离，可独立维护和扩展
- 每套模板有独立的视觉规范（配色、字体、背景、装饰等）
- 支持新增模板和切换模板

### FR-02 AI 幻灯片生成

- 用户先选择模板
- 用户通过自然语言描述演示需求
- AI 套用所选模板，根据需求组合页面（封面 → 目录 → 内容 → 封底）
- AI 填充具体内容并保持模板风格一致
- 流式输出，逐页生成不等全部完成

### FR-03 对话式迭代

- 通过对话逐页修改内容
- 支持调整页面布局（如从单栏改为两栏）
- 支持增删页面、调整页序
- 支持调整配色、字体等视觉元素
- 编辑历史可 undo / redo

### FR-04 LLM API Key 管理

- 支持多种 AI 服务提供商：
  - **OpenAI 兼容接口**：OpenAI / DeepSeek / 智谱等，统一适配
  - **原生接口**：Anthropic Claude / Google Gemini，发挥各自原生能力（system prompt / tool use / 结构化输出）
- 每个 provider 独立配置 API Key、模型、可选 baseUrl
- 用户可在 provider 间自由切换
- API Key 在生产环境服务端加密存储；本地开发可放浏览器侧便捷调试
- LLM 调用统一在服务端进行，客户端不携带敏感凭据

### FR-05 导出

- 导出为 PDF
- 导出为 PNG 图片序列
- 导出为 PPTX（探索性，看实现成本）
- 本地演示模式（presenter mode）
- 导出历史记录可重新下载

### FR-06 用户账号与登录

- 邮箱 + 密码注册 / 登录 / 登出
- 密码安全哈希，明文不落 DB 或日志
- Session 有撤销能力（服务端 stateful）
- 刷新页面保留登录态

### FR-07 多 Deck 管理

- 登录后查看自己的 deck 列表（标题、状态、更新时间）
- 新建、重命名、软删（status='deleted'）
- 打开某 deck 进入编辑页，切换 deck 时系统自动处理实例内容切换
- Deck 归属私有（分享链接见 FR-12）

### FR-08 版本历史与回滚

- 每次保存自动追加版本记录（append-only）
- 版本时间线面板展示所有历史版本
- 一键回滚到任意历史版本，保留完整时间线
- 回滚后对话历史保留连续，AI 能理解"用户改主意了"的语境

### FR-09 单实例并发控制

- 同一时刻只允许一个用户占用编辑实例
- 第二个用户尝试进入已占用 deck 时显示等待页：当前占用者、锁定时长、最近活跃时间
- 等待页轮询锁状态，释放后自动跳转
- 占用中客户端定期发送心跳
- 主动释放（点击"结束编辑" / 窗口关闭）立即放行
- 心跳超时自动判定释放，防止用户强退留下死锁

### FR-10 部署与上线

- 单实例上线：作为内部 MVP 串行使用
- 多实例上线：用户并行编辑各自 deck 互不干扰
- 部署形态：自有服务器 / docker compose / 私有化部署，**不依赖第三方 BaaS**
- 上线前全量安全 audit 通过

### FR-11 导入

- Markdown 导入（粘贴或上传或 URL 拉取）
- PPTX 导入（探索性，看转换质量）
- 导入预览页让用户确认后才落库
- 失败场景给出可操作的提示（行号 / 原文保留）

### FR-12 分享链接

- 用户可对 deck 生成只读分享链接
- 分享页不需要登录可访问
- 分享访问不占用编辑实例
- 链接可设置过期时间、撤销

### FR-13 MCP 工具生态

- 平台预置常用 MCP 服务（联网搜索、网页读取、视觉、知识库等），用户填入凭据即可使用
- 用户可自定义接入第三方 MCP 服务
- 工具调用结果显式反馈到对话流，可视化每一步

### FR-14 模板分层架构

- 模板分两层：
  - **第一层**：模板独有的 5 个核心 layout（封面、封底、目录、章节标题、内容页骨架）
  - **第二层**：所有模板共享的内容部件（布局、图表、媒体），只读设计 token 自动配色
- 切换模板时内容页**字节级保留**——只有视觉随新模板变化，业务内容不变
- AI 生成内容页时只决定语义（"用田字格放四个特性"），具体视觉由当前模板 token 决定

### FR-15 模板生态远期

> 远期愿景，依赖用户基数与平台成熟度，详见 [`vision.md`](./vision.md#远期愿景模板生态系统)。

- 模板创作工具链：脚手架 + 本地预览 + 规范校验 + 缩略图生成
- 模板市场：创作者上架 → 审核 → 用户购买 → 收入分成
- 设计 token 跨模板通用规范，保证公共组件在任何模板下视觉协调

## 非功能需求

### NFR-01 性能

- AI 生成完整演示文稿 < 30 秒
- 幻灯片渲染流畅，无明显卡顿
- 流式输出，不阻塞用户感知
- 可承载 ~50 人规模的内部团队并发使用（编辑侧），分享/查看侧无并发上限

### NFR-02 安全

- API Key 服务端加密存储；用户密码安全哈希
- Session cookie 走 HttpOnly + SameSite + Secure（生产）
- 用户内容不经过非必要第三方服务（除 AI API 调用外）
- 敏感信息（DB 密码、master key、session secret）通过环境变量下发，**绝不进 git**
- 多用户场景下严格的所有权检查，不跨用户访问

### NFR-03 可维护性

- 模板与代码分离，非开发人员也可维护模板
- 清晰的项目结构与文档分层（vision / requirements / roadmap / plans / CLAUDE.md）
- 模板新增不需要改动核心代码

### NFR-04 兼容性

- 主流桌面浏览器（Chrome / Firefox / Safari / Edge）
- 移动端可查看（响应式），编辑端不强求
