# Lumideck 项目愿景

> 本文档定位：**产品形态 + 商业模式畅想**。不涉及技术细节。
>
> 技术栈、目录结构等工程信息见 [`CLAUDE.md`](../../CLAUDE.md)；具体功能点见 [`requirements.md`](./requirements.md)；落地排期见 [`roadmap.md`](./roadmap.md)。

## 项目背景

公司内部经常需要制作各类演示文稿（汇报、技术分享、项目复盘等），但每次都要从零开始排版设计，效率低且风格不统一。市面上的 PPT 工具虽然强大，但缺少与公司模板深度集成的 AI 辅助能力。

## 核心愿景

**通过 AI 对话，基于模板一键生成专业演示文稿。**

用户选择模板后描述需求，AI 套用模板结合用户内容组合页面、填充内容，生成符合视觉规范的演示文稿，并支持持续对话迭代。

## 目标用户

| 阶段        | 目标用户                                       | 场景画像                                                                         |
| ----------- | ---------------------------------------------- | -------------------------------------------------------------------------------- |
| 早期（已验证） | 公司内部员工                                | 汇报、技术分享、项目复盘——有现成模板规范，但每次排版都要重做                         |
| 中期           | 行业垂直客户的小团队                         | 教育、咨询、内容创作团队——已有自己的视觉规范但缺乏 AI 辅助能力                        |
| 远期           | 任何需要频繁产出演示稿的个人 / 团队         | "选个我喜欢的模板 + 跟 AI 聊几句 = 一份能用的 PPT"                                 |

## 核心场景

1. **选模板 → 对话 → 生成**：用户从模板库挑选风格 → 描述演示主题与内容要求 → AI 在所选模板内组合页面、填充内容
2. **迭代优化**：通过对话逐页调整（"把第 3 页改成两栏"），或在编辑器内直接微调
3. **历史回溯**：每次保存自动入版本历史，任何版本一键回滚，不丢上下文
4. **多 deck 管理**：登录后管理自己所有的 deck，归档、删除、切换
5. **导出与分享**：导出 PDF / 图片用于线下；分享只读链接给他人查看，不需要登录
6. **导入起步**：已有 Markdown / PPTX 一键转 deck，降低冷启动成本

## 产品形态演进

### 单实例阶段：内部串行使用

最早期版本上线后只跑单实例，同一时刻只允许一个用户编辑。其他用户登录后看到等待页，原占用者主动释放或长时间无操作后自动放行。这一阶段足以覆盖小团队内部 MVP 验证。

### 多实例阶段：并行编辑 + 静态分享

平台演进为"每位活跃用户一个独立编辑环境 + 只读分享链接零运行开销"的混合架构：

- **编辑路径**：用户进入编辑页时分配独立资源，HMR 体验保留
- **分享路径**：保存即触发构建，产物静态托管；只读访问无并发上限

这让"每人一份 deck"真正成立，编辑侧保留实时预览体验，查看 / 分享侧水平可扩展。

## 商业模式畅想

> 本节是**畅想性的远期路线**，不是已定盘的商业计划。落地节奏取决于用户基数、产品成熟度、市场验证结果。

### 阶段 1：内部生产力工具（当前）

**定位**：公司内部基础设施，零费用。覆盖 ~50 人规模的团队并发使用。

**目标**：验证"AI + 模板"心智在真实工作场景里能不能省时间、能省多少时间；积累内部反馈打磨产品。

### 阶段 2：行业垂直 SaaS

**定位**：把内部验证的产品经验封装成对外 SaaS，目标行业客户的小团队（教育 / 咨询 / 内容创作）。

**形态**：

- **订阅制**：按席位或按 deck 数量计费
- **核心价值主张**：一份与客户视觉规范深度耦合的"AI PPT 助手"，而不是又一个通用 AI 写作工具
- **私有化部署**选项：对数据敏感型客户（金融 / 医疗 / 法务）提供独立部署
- **入驻模板**：平台预置 5-10 套行业通用模板（教育、咨询、产品发布、技术分享等），客户开箱即用

### 阶段 3：模板创作者市场

**定位**：从"工具"演变为"生态"——把"模板生产"开放给第三方设计师。

**核心痛点**：每家公司都要单独定制模板，模板生产成本高且分散。如果有一个市场让设计师集中产出、用户集中消费，双方都受益。

**形态**：

- **创作者侧**：设计师注册入驻 → 用平台提供的工具链产出模板 → 上架审核 → 发布
- **用户侧**：在 lumideck 内浏览市场、试用、付费购买、加入自己账号
- **平台侧**：审核 + 抽成（如 30% 平台分成 / 70% 创作者分成）
- **价值飞轮**：好模板吸引用户 → 用户基数吸引创作者 → 更多模板覆盖更多场景 → 更大用户基数

**前提条件**：

- 阶段 2 的 SaaS 模型跑通，有稳定用户基数
- 模板生态架构在工程层已落地（详见下文「远期愿景：模板生态系统」）
- 法务 / 财务 / 审核流程跑得通（创作者协议 / 收入分成 / 内容合规）

## 远期愿景：模板生态系统

### 为什么需要"生态系统"而不是"内置模板"

只靠平台自己手写模板，能覆盖的视觉风格有限、迭代慢、无法应对长尾需求。当用户从内部团队扩展到行业客户，再扩展到通用市场时，"模板供给"会成为产品的核心瓶颈。让生态来生产模板，而不是靠一个团队，是平台型产品的必经之路。

### 核心心智：模板分两层

> 该心智的工程实现细节见对应实施计划，本节只描述产品形态。

一份完整演示文稿无非 5 类页面：**封面 / 封底 / 目录 / 章节标题 / 内容页**。前 4 类是模板的"门面"，视觉差异极大；最后一类内容页占了 80% 篇幅，但每页结构其实都很相似——标题 + 装饰边角 + 大块留白让用户填具体内容。

基于这个观察，模板拆成两层：

- **第一层（模板独有）**：5 个 layout 决定模板的"门面"——封面、封底、目录、章节标题、内容页骨架。这一层每套模板必须自己定制，体现品牌差异
- **第二层（所有模板共享）**：内容页里要塞的具体部件——两栏 / 三栏 / 田字格 / 九宫格 / 各类图表 / KV 列表 / 数据卡片等——下沉为公共组件库，所有模板共用，**只通过设计 token 自动配色**

### 这套架构的产品价值

- **模板生产门槛降一个数量级**：第三方设计师只需交付 5 个 layout + 一份配色规范，不用重写一堆 chart 和布局组件
- **模板切换无损**：用户切模板时内容完全保留，只是配色 / 字体 / 装饰随新模板变化——这是模板市场体验的关键（用户敢于试用付费模板）
- **AI 创作更聚焦**：AI 决定"这页用田字格放四个核心特性"这类语义层面的事，具体视觉由当前模板的 token 决定，不用为每个模板重训 prompt

### 配套生态设施

- **模板创作工具链**：脚手架命令一键起新模板、本地预览、规范校验、缩略图生成——把"做一套模板"的复杂度降下来
- **模板市场**：创作者上架 → 平台审核 → 用户浏览 / 试用 / 购买 → 收入分成
- **设计 token 规范**：跨模板通用的色板 / 字体 / 形状 / 阴影规范，确保公共组件在任何模板下视觉协调

## 成功标准

| 维度       | 目标                                                                |
| ---------- | ------------------------------------------------------------------- |
| 体验       | 从描述到生成完整演示稿 < 30 秒；用户可通过对话自由调整任何页面            |
| 视觉       | 基于模板生成的幻灯片符合所选模板的视觉规范，无需用户手动美化              |
| 模板覆盖   | 早期支持至少 3 套高质量模板；远期生态化后由市场决定供给                 |
| 历史可控   | 任何编辑都可回溯，回滚不丢对话上下文                                   |
| 部署       | 支持本地运行 / 内部单实例 / 多实例 SaaS / 私有化部署四种形态              |
| 数据安全   | 用户内容不外泄（除 AI API 调用外），模板创作者上传内容隔离                |
