# Big-PPT 项目愿景

## 项目背景

公司内部经常需要制作各类演示文稿（汇报、技术分享、项目复盘等），但每次都要从零开始排版设计，效率低且风格不统一。市面上的 PPT 工具虽然强大，但缺少与公司模板深度集成的 AI 辅助能力。

## 核心愿景

**通过 AI 对话，基于公司模板一键生成专业演示文稿。**

用户选择模板套后描述需求，AI 套用模板结合用户内容组合页面、填充内容，生成符合公司视觉规范的 Slidev 演示文稿。

## 目标用户

- 公司内部员工
- 需要制作汇报、分享、复盘等演示文稿的人员
- 不熟悉设计但希望产出专业级幻灯片的开发者

## 核心场景

1. **选择模板**：用户从可用模板套中选择合适的模板
2. **绑定 API Key**：首次使用时配置 AI 服务密钥（Phase 5 起由 agent 后端 AES-256-GCM 加密存储）
3. **对话式生成**：描述演示主题和内容要求，AI 套用所选模板生成完整幻灯片
4. **迭代优化**：本地 dev 模式下通过 Slidev 前端实时编辑 md 内容，或通过对话调整
5. **版本回溯**（Phase 5）：从历史版本时间轴挑一条旧版内容一键回滚，保留完整编辑轨迹
6. **多 deck 管理**（Phase 5）：登录后看到自己所有 deck 的列表、状态、创建时间；可新建、归档、删除。单实例模式下同一时刻只有一人占用 Slidev，其他用户登录后看到等待页，原占用者主动释放或心跳超时后自动放行
7. **单实例上线部署**（Phase 5.5）：MVP 版本单实例部署到服务器，内部小范围使用
8. **多人并行使用**（Phase 6）：Slidev 进程池升级后，多个用户同时编辑各自的 deck 互不干扰；deck 可通过分享链接发给他人只读查看
9. **导出带走**（Phase 7）：本地演示、导出 PDF/PNG/PPTX
10. **导入起步**（Phase 8）：已有 Markdown / PPTX 一键转 deck

## 多用户架构的关键抉择

Slidev 本质是一个 Vite dev server 吃一份 `slides.md` 启动，**原生不支持多文件并行**。Big-PPT 的设计把这个问题分两步解决：

### Phase 5 过渡态：单实例 + 占用锁

Phase 5/5.5 不上进程池，Slidev 只跑一个实例。多 deck 通过"切换 deck 时 agent 改写那份 slides.md"实现。同一时刻只有一个用户占用（全局 `slidev_lock` 表 + 30s 心跳 + 5 分钟超时自动释放），其他人登录后看到等待页并轮询。好处是实现成本低、部署简单，足以支撑内部小范围使用和 MVP 验证。

### Phase 6 终态：进程池 + 静态分享（A + B 混合）

- **编辑路径**：agent 维护 Slidev 子进程池（LRU，上限由容量 spike 结果决定），用户进编辑页时 on-demand spawn，空闲 5 分钟回收。每个活跃用户占一个进程（预估内存 200-300MB）
- **分享 / 查看路径**：用户保存时触发 `slidev build`，产物静态托管；只读访问不占实例，水平可扩展

这让"每人一份 deck"真正成立：编辑侧保留 Slidev 原生的 HMR 体验，查看 / 分享侧零运行开销。

## 技术栈

| 层面           | 技术                                             |
| -------------- | ------------------------------------------------ |
| 演示框架       | Slidev                                           |
| UI 框架        | Vue 3                                            |
| 构建工具       | Vite                                             |
| CSS 引擎       | UnoCSS                                           |
| AI 集成        | OpenAI 兼容接口（智谱 / DeepSeek / OpenAI 等）   |
| 后端           | Hono + Node.js（`packages/agent`）               |
| 数据库         | MySQL + Drizzle ORM（Phase 5+；复用既有 MySQL 实例） |
| 密码哈希       | bcrypt rounds=10（Phase 5+）                     |
| Session        | HttpOnly Cookie + 服务端 sessions 表（Phase 5+） |
| 包管理器       | pnpm + Turborepo                                 |
| Monorepo       | pnpm workspace（`packages/*`）                   |
| 单实例并发控制 | `slidev_lock` + 心跳 + 等待页（Phase 5）          |
| 多 deck 运行时 | Slidev 子进程池（Phase 6+）                      |
| 首次部署       | 单实例：域名 + HTTPS + systemd/compose（Phase 5.5） |
| 多实例部署     | 反代按 session 路由 + 灰度切换（Phase 6 尾段）    |

## 成功标准

- 基于模板生成的幻灯片符合公司视觉规范
- 从描述到生成完整演示文稿 < 30 秒
- 支持至少 3 套公司模板
- 用户可通过对话自由调整任何页面
- 历史版本随时可回溯（Phase 5+）
- 单实例上线后支持内部串行使用（Phase 5.5）
- 支持 ~50 人规模的内部团队并发使用（同时活跃编辑上限由容量 spike 决定，查看侧无上限，Phase 6+）
- 支持本地运行和云端部署
- 支持 PDF/图片序列导出（Phase 7+），Markdown 导入（Phase 8+）
